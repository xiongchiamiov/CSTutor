2.6. Units of Saveability

This section describes how and where data is stored when using the CSTutor. The database for the CSTutor will be organized into the CSTutor databse which contains a user database and a course database. The user database will contain all the users registered for the CSTutor. In addition, the course database will contain all the information related to a course. Furthermore, the CSTutor database will not be installed to a predetermined location. This location will be determine when during installation of the CSTutor.

2.6.1 Adding a User to the Class Roster

Adding a user to the class roster is done through the Roster Management page as described in Section 2.3.4.1.

When a user is added to the roster, that user is only saved into the local workspace until the Submit button is clicked at the bottom of the Roster Management page.

Once the Submit button is clicked, the changes are then saved to the user database.

2.6.2 Removing a User from the Class Roster

Removing a user from the class roster is done through the Roster Management page as described in Section 2.3.4.2.

When a user is removed from the roster, that user is only removed from the local workspace until the Submit button is clicked at the bottom of the Roster Management page.

Once the Submit button is clicked, the changes are then saved to the user database.

2.6.3 Editing a User's Permissions in the Class Roster

Editing a user's permissions in the class roster is done through the Roster Management page as described in Section 2.3.4.3.

When a user's permissions are eidting, those permissions are only saved in the local worksapce until the Submit button is clicked at the bottom of the Roster Management page.

Once the Submit button is clicked, the changes are then saved to the user database.

2.6.4 Managing User Requests to join the Class Roster

Managing user requests to join the class roster is done through the Roster Management page as described in Section 2.3.4.4.

When an instructor confirms or denys a request to join the roster, the class roster changed is only reflected in the local workspace until the Submit button is clicked at the bottom of the Roster Management page.

Once the Submit button is clicked, the changes are then saved to the user database.

2.6.5 Importing a Class Roster

Importing a class roster from a file is done through the Roster Management page as described in Section 2.3.4.5.

When an instructor imports a class roster from a file, the class roster is only modified in the local workspace until the Submit button is clicked at the bottom of the Roster Management page.

Once the Submit button is clicked, the changes are then saved to the user database.
